Specific format preferences

In addition to the global settings you can set specific preferences for every handled file format.

In the file type Specific settings there is a list of all file types supported by the quicklook extension.
For an explanation on the behavior of file types click here.

The grayout items on the list are those currenty managed by others UTI. Clicking on the exclamation mark icon you will show which UTI is used for a given file extension.

For an explanation of the various parameters see the page of global settings.

In the specific settings you can also set a preprocessor utility. The preprocessor handle the source file before it is elaborated by highlight.
Preprocessor can be any cli utility or shell script which receives as the last parameter the name of the file to be processed and output to stdout the data to pass to highlight.

Preprocessor can be useful to convert a format into another (eg from binary to text), decompile, pretty printingâ€¦

The excution of the preprocessor is made inside the same env of the script that handle highlight. For a list of environments plese see the file colorize.sh in Syntax Highlight.app/Contents/XPCServices/Syntax Highlight XPC Service.xpc/Contents/Resources/

Specifying a custom preprocessor disables the predefined preprocessors:

file typedefault proprocessoraction
.class /usr/local/bin/jad -ff -dead -noctor -p -t Decompile the class file, require the installation on the free jad decompiler.
.ascr | .scpt /usr/bin/osadecompile Decompile an applescript applet.
.plist /usr/bin/plutil -convert xml1 -o - Convert a binary plist to xml format.
all other files cat Simply output the contents of the file.
